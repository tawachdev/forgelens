Context limits are not the real enemy. Bad focus is.

A bigger context window still fails when the agent starts in the wrong place. ForgeLens turns static repo signals into a practical read order.

01

Risk-first maps

Admin routes, API handlers, server actions, database clients, and schema files get ranked before low-risk UI files.

02

Drift detection

Compare a baseline report with the current repo and see what changed around auth, data, env, and focus files.

03

Safe local output

Scans run locally. Env reports list file names and key names only, never secret values.

One workflow before every serious AI edit.

Generate context, save a baseline, compare drift, then give the AI a clean read order.

1

Scan

Run forgelens scan --format all to generate Markdown and JSON context.

2

Save baseline

Use forgelens baseline save when the repo map is known-good.

3

Detect drift

Run forgelens drift --from latest or compare main..HEAD.

4

Guide the agent

Start with AI_COMPACT_CONTEXT, then read drift and risk reports.
